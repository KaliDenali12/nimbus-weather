# Nimbus Weather App — Configuration Reference

> Generated by DevOps audit on 2026-03-18.
> Documents all configuration constants, their locations, and rationale.

---

## Overview

Nimbus is a **zero-config** client-side SPA. There are no environment variables, no `.env` files, and no secrets. All configuration is hardcoded as constants in source files. This is intentional: the app uses only the free, keyless Open-Meteo API and deploys as a static site to Netlify.

## API Configuration

**File**: `src/lib/api.ts`

| Constant | Value | Line | Purpose |
|----------|-------|------|---------|
| `GEOCODING_URL` | `https://geocoding-api.open-meteo.com/v1/search` | 3 | City search endpoint |
| `FORECAST_URL` | `https://api.open-meteo.com/v1/forecast` | 4 | Weather forecast endpoint |
| `GEOCODING_RESULT_LIMIT` | `8` | 5 | Max search results returned |
| `FORECAST_DAYS` | `6` | 6 | Today + 5 forecast days |
| `MIN_SEARCH_QUERY_LENGTH` | `2` | 7 | Minimum chars before API call |
| `MAX_SEARCH_QUERY_LENGTH` | `200` | 8 | Maximum search input length |
| `GEOCODING_CACHE_TTL` | `5 min` (300,000ms) | 9 | In-memory geocoding cache lifetime |
| `GEOCODING_CACHE_MAX_ENTRIES` | `50` | 10 | Max cached geocoding queries |
| `FORECAST_CACHE_TTL` | `5 min` (300,000ms) | 11 | In-memory forecast cache lifetime |
| `FORECAST_CACHE_MAX_ENTRIES` | `10` | 12 | Max cached forecast results |
| `FETCH_TIMEOUT_MS` | `10,000ms` (10s) | 13 | Timeout for all API fetch calls |

**Notes**:
- Cache is in-memory only (Map objects). Cleared on page refresh.
- FIFO eviction when cache is full: oldest entry removed first.
- All fetch calls use `AbortSignal.timeout()` for timeout enforcement.

## Storage Configuration

**File**: `src/lib/storage.ts`

| Constant | Value | Line | Purpose |
|----------|-------|------|---------|
| `STORAGE_KEY` | `nimbus-preferences` | 3 | localStorage key name |
| `MAX_RECENT_CITIES` | `5` | 4 | Maximum saved recent cities |
| `MAX_CITY_NAME_LENGTH` | `200` | 5 | Validation: max city name length |

**Stored schema** (`nimbus-preferences`):
```typescript
{
  unitPreference: 'celsius' | 'fahrenheit'
  darkModeEnabled: boolean
  recentCities: Array<{
    name: string      // 1-200 chars
    lat: number       // -90 to 90
    lon: number       // -180 to 180
    country: string
  }>  // max 5, FIFO with lat/lon dedup
}
```

## Why No Environment Variables?

1. **No secrets** — Open-Meteo requires no API keys
2. **No backend** — All logic runs in the browser
3. **Single environment** — Same code runs in dev and production
4. **No build-time config** — All values are static constants
5. **Simplicity** — Adding env vars for constants that never change adds complexity without benefit
